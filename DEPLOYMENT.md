# 📦 Развертывание на офлайн-компьютере

Этот документ описывает как перенести проект на компьютер без доступа к интернету.

---

## 🟢 ЭТАП 1: Подготовка (на ПК с интернетом)

### 1. Собери Docker-образ

Открой терминал в папке проекта и выполни:

```bash
# Собери образ
docker build -t annotator:v1 .

# Сохрани образ в файл (для флешки)
docker save annotator:v1 -o annotator.tar
```

### 2. Подготовь файлы для флешки

Скопируй на флешку:

| Файл/Папка | Описание |
|------------|----------|
| `annotator.tar` | Docker-образ с проектом |
| `Dockerfile` | Инструкции для Docker |
| `docker-compose.yml` | Конфигурация запуска |
| `requirements.txt` | Зависимости Python |
| `DEPLOYMENT.md` | Эта инструкция |

### 3. Скачай установщики

| Программа | Где скачать |
|-----------|-------------|
| **Docker Desktop** | https://www.docker.com/products/docker-desktop/ |
| **Ollama** | https://ollama.com/download |

### 4. Скачай модели Ollama

Выполни команды для скачивания нужных моделей:

```bash
# Пример для легких моделей (подходят для слабых ПК)
ollama pull qwen2.5:1.5b
ollama pull qwen2.5:3b
ollama pull llama3.2:1b
ollama pull llama3.2:3b
```

### 5. Скопируй модели на флешку

Найди папку с моделями и скопируй её на флешку:

**Windows:**
```
C:\Users\ТвоёИмя\.ollama\models\
```

**Linux/Mac:**
```
~/.ollama/models/
```

---

## 🔴 ЭТАП 2: Установка (на офлайн-компьютере)

### 1. Установи Docker Desktop

1. Запусти `Docker Desktop Installer.exe` с флешки
2. Следуй инструкциям установщика
3. **Перезагрузи компьютер** (обязательно!)
4. Запусти Docker Desktop и дождись запуска

### 2. Установи Ollama

1. Запусти `OllamaInstaller.exe` с флешки
2. Следуй инструкциям установщика
3. Ollama запустится автоматически (иконка в трее)

### 3. Перенеси модели

Скопируй папку `models` с флешки в папку Ollama:

**Windows:**
```
C:\Users\ИмяПользователя\.ollama\models\
```

**Linux:**
```
~/.ollama/models/
```

### 4. Проверь модели

Открой терминал (PowerShell или CMD) и выполни:

```bash
ollama list
```

Должен появиться список скачанных моделей.

---

## 🚀 ЭТАП 3: Запуск проекта

### Способ 1: Через Docker Compose (рекомендуется)

1. Скопируй файлы проекта на компьютер:
   - `docker-compose.yml`
   - `annotator.tar`

2. Открой терминал в папке с файлами

3. Загрузи образ:
   ```bash
   docker load -i annotator.tar
   ```

4. Запусти проект:
   ```bash
   docker-compose up -d
   ```

5. Открой браузер:
   ```
   http://localhost:8000
   ```

### Способ 2: Через Docker Run

1. Загрузи образ:
   ```bash
   docker load -i annotator.tar
   ```

2. Запусти контейнер:
   ```bash
   docker run -d -p 8000:8000 --name annotator annotator:v1
   ```

3. Открой браузер:
   ```
   http://localhost:8000
   ```

---

## ⚙️ ЭТАП 4: Настройка в веб-интерфейсе

1. Открой `http://localhost:8000`

2. В разделе **"Настройки моделей"**:
   - Выбери режим **Local**
   - Нажми **"Применить и перезапустить сервер"**

3. После перезапуска:
   - Нажми **"Обновить список моделей"**
   - Выбери модели для перевода, аннотирования и проверки

4. Готово! Можно работать.

## 🔧 Полезные команды

| Команда | Описание |
|---------|----------|
| `docker ps` | Показать запущенные контейнеры |
| `docker logs annotator` | Посмотреть логи проекта |
| `docker-compose down` | Остановить проект |
| `docker-compose up -d` | Запустить проект |
| `ollama list` | Показать доступные модели |
| `ollama serve` | Запустить Ollama (если не запущен) |

## ⚠️ Возможные проблемы

### 1. Ошибка "Cannot connect to Docker daemon"
**Решение:** Убедись, что Docker Desktop запущен.

### 2. Ошибка "Connection refused" к Ollama
**Решение:** 
- Убедись, что Ollama запущен (иконка в трее)
- Проверь `ollama list` в терминале

### 3. Модели не отображаются
**Решение:**
- Перезапусти Ollama
- Проверь что папка `models` скопирована правильно

### 4. Страница не открывается
**Решение:**
- Проверь что контейнер запущен: `docker ps`
- Проверь логи: `docker logs annotator`
- Попробуй `http://127.0.0.1:8000` вместо `localhost`

---

## 📞 Поддержка

Если возникли проблемы:
1. Проверь логи Docker: `docker logs annotator`
2. Проверь логи Ollama (в трее → View Logs)
3. Убедись что порты не заняты другими программами
