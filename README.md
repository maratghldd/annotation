# Документ-анализатор с модульным Ollama pipeline

Веб-приложение для автоматической аннотации документов с использованием локальной нейросети Ollama.

## Особенности

- **Многоэтапный pipeline** с разделением задач на отдельные модели:
  1. **Модель перевода** — перевод текстов с украинского на русский
  2. **Модель аннотирования** — создание подробного описания (мини-пересказа) документа
  3. **Модель проверки** — валидация и уточнение сгенерированной аннотации

- **Гибкая конфигурация**:
  - **Обязательный выбор моделей** для каждой задачи через веб-интерфейс
  - Включение/отключение отдельных этапов pipeline
  - Поддержка произвольных моделей из Ollama
  - **Валидация** — запуск без выбранных моделей невозможен

- **Поддерживаемые форматы**:
  - `.docx` — Microsoft Word
  - `.doc` — конвертация через LibreOffice
  - `.pdf` — извлечение текста

- **Функции интерфейса**:
  - Анализ отдельных файлов или целых папок
  - Отслеживание прогресса в реальном времени через WebSocket
  - Редактирование сгенерированных названий
  - Экспорт результатов в CSV/Excel/TXT
  - Отмена выполняемых задач с очисткой созданных файлов

## Установка

1. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```

2. Установите LibreOffice для конвертации `.doc` файлов:
   - Ubuntu/Debian: `sudo apt install libreoffice`
   - macOS: `brew install --cask libreoffice`
   - Windows: скачайте с https://www.libreoffice.org

3. Убедитесь, что Ollama доступна по адресу `https://ollama.k2.iksi.edu` (или настройте через переменные окружения)

## Запуск

```bash
python run.py
```

Или напрямую через uvicorn:
```bash
uvicorn app:app --reload --host 0.0.0.0 --port 8000
```

Откройте браузер по адресу: `http://localhost:8000`

## Конфигурация

### Переменные окружения

| Переменная | Описание | Значение по умолчанию |
|------------|----------|----------------------|
| `OLLAMA_URL` | URL Ollama сервера | `https://ollama.k2.iksi.edu` |
| `OLLAMA_VERIFY_SSL` | Проверка SSL сертификатов | `false` |
| `ENABLE_TRANSLATION` | Включить этап перевода | `true` |
| `ENABLE_ANNOTATION` | Включить этап аннотирования | `true` |
| `ENABLE_REVIEW` | Включить этап проверки | `true` |

> ⚠️ **Важно:** Модели для каждой задачи **выбираются пользователем** через веб-интерфейс перед запуском анализа. Значения по умолчанию не установлены — это предотвращает случайное использование неподходящих моделей.

### Пример запуска с локальной Ollama

```bash
OLLAMA_URL=http://localhost:11434 \
python run.py
```

> 📌 Модели выбираются в веб-интерфейсе после запуска

## API Endpoints

### Анализ
- `POST /api/analyze-folder` — Запуск анализа папки
- `POST /api/analyze-file` — Загрузка и анализ файла
- `POST /api/cancel-task` — Отмена выполняемой задачи

### Статус и результаты
- `GET /api/status/{task_id}` — Получение статуса задачи
- `GET /api/results/{task_id}` — Получение результатов
- `GET /api/files/{task_id}/{filename}` — Получить обработанный файл

### Конфигурация
- `GET /api/models` — Получить список доступных моделей и текущую конфигурацию
- `POST /api/test-connection` — Проверить подключение к Ollama

### Дополнительные функции
- `POST /api/regenerate-title` — Перегенерировать название для файла
- `PUT /api/update-title` — Обновить название вручную
- `GET /api/export/{task_id}?format={csv|excel|txt}` — Экспорт результатов
- `GET /api/browse?path={path}` — Обзор файловой системы
- `WebSocket /ws/logs/{task_id}` — Лог выполнения в реальном времени

## Использование

### Анализ папки

1. Введите путь к исходной папке с документами
2. Укажите папку для сохранения результатов
3. Настройте модели (по умолчанию используются значения из конфигурации)
4. Нажмите "Запустить анализ папки"
5. Следите за прогрессом в логе
6. После завершения просмотрите результаты в таблице

### Анализ одного файла

1. Переключитесь на режим "Один файл"
2. Перетащите файл или выберите через диалог
3. Дождитесь завершения анализа
4. Результат отобразится в панели и таблице

### Настройка моделей

1. Раскройте секцию "Настройки моделей"
2. Выберите модели из списка для каждой задачи:
   - **Модель перевода** — для перевода с украинского
   - **Модель аннотирования** — для создания описания
   - **Модель проверки** — для валидации результата
3. Включите/отключите отдельные этапы pipeline
4. Нажмите "Обновить список моделей" для получения актуального списка из Ollama

## Примеры использования

### Быстрый старт с настройками по умолчанию

```bash
python run.py
# Откройте http://localhost:8000
# Укажите папку с документами и запустите анализ
```

### Использование локальной Ollama

```bash
# Установите и запустите Ollama
ollama pull llama2
ollama pull mistral
ollama pull phi3

# Запустите приложение с локальной Ollama
OLLAMA_URL=http://localhost:11434 \
OLLAMA_TRANSLATE_MODEL=llama2 \
OLLAMA_ANNOTATE_MODEL=mistral \
OLLAMA_REVIEW_MODEL=phi3 \
python run.py
```

### Отключение этапов pipeline

```bash
# Только аннотирование без перевода и проверки
ENABLE_TRANSLATION=false \
ENABLE_REVIEW=false \
python run.py
```
